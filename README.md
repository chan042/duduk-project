## 🌟 주요 기능 (Key Features)

### 1. 🤖 Gemini 기반 AI 소비 코칭

단순한 통계가 아닙니다. **Google Gemini API**를 활용하여 당신의 소비 패턴을 심층 분석합니다.

* **개인화 피드백**: "지난달보다 카페 지출이 20% 늘었어요. 텀블러를 사용해보는 건 어떨까요?" 같은 구체적인 행동 제안
* **절약 시뮬레이션**: 코칭을 따랐을 때 모을 수 있는 예상 금액 제시
* **감정 분석**: 소비 내역에 따른 칭찬과 위로의 메시지 제공

### 2. 🐾 나만의 '두둑이' 키우기

절약하면 캐릭터가 성장합니다! 4가지 귀여운 캐릭터 중 하나를 선택해 파트너로 삼으세요.
| 🐱 고양이 (Cat) | 🐶 강아지 (Dog) | 🐹 햄스터 (Hamster) | 🐑 양 (Sheep) |
| :---: | :---: | :---: | :---: |
| <img src="frontend/public/images/characters/char_cat/face_happy.png" width="80"> | <img src="frontend/public/images/characters/char_dog/face_happy.png" width="80"> | <img src="frontend/public/images/characters/char_ham/face_happy.png" width="80"> | <img src="frontend/public/images/characters/char_sheep/face_happy.png" width="80"> |

> *캐릭터는 사용자의 소비 상태(절약 성공, 과소비 등)에 따라 표정이 실시간으로 변화합니다!*

### 3. 🏆 게이미피케이션 챌린지

지루한 돈 모으기는 그만. 게임처럼 즐기세요.

* **자동 판정 시스템**: 목표 금액과 기간을 설정하면 시스템이 지출 내역을 분석해 성공 여부를 자동 판정
* **다양한 모드**:
* **두둑 챌린지**: "일주일 무지출 도전", "커피값 줄이기" 등 공식 미션
* **유저 챌린지**: 내가 직접 만드는 커스텀 미션


* **보상**: 챌린지 성공 시 포인트 지급 및 캐릭터 꾸미기 아이템 획득

### 4. 📅 스마트 가계부 & 예산 관리

* **예산 스냅샷**: 매일 자정, 남은 일수와 잔액을 계산하여 '오늘 쓸 수 있는 돈'을 알려줍니다.
* **무지출 도장**: 지출이 없는 날은 '무지출 확인' 스탬프 쾅!
* **고정 지출 분리**: 월세, 구독료 등 숨만 쉬어도 나가는 돈을 따로 관리하여 정확한 가용 자산을 파악합니다.

## 🏗️ 시스템 아키텍처 (Architecture)

프로젝트는 **Docker Compose**를 기반으로 3개의 마이크로서비스가 유기적으로 연결되어 있습니다.

```mermaid
graph LR
    A[Client (Browser)] -- Port 3000 --> B(Frontend: Next.js)
    A -- Port 8000 --> C(Backend: Django API)
    C -- AI Request --> D[Google Gemini API]
    C -- SQL Query --> E[(Database: PostgreSQL)]

```

| 서비스 | 기술 스택 | 설명 |
| --- | --- | --- |
| **Frontend** | Next.js 16, React 19 | App Router 기반의 반응형 웹 애플리케이션 |
| **Backend** | Django 5, DRF | RESTful API 서버, JWT 인증, 비즈니스 로직 처리 |
| **Database** | PostgreSQL 15 | 사용자 데이터 및 트랜잭션, 로그 저장소 |

## 🚀 시작 가이드 (Getting Started)

### 1. 프로젝트 설치

```bash
git clone https://github.com/your-username/duduk-project.git
cd duduk-project

```

### 2. 환경 변수 설정

최상위 경로에 `.env` 파일을 생성하고 아래 내용을 채워주세요.

```properties
# --- Database Configuration ---
POSTGRES_DB=duduk_db
POSTGRES_USER=duduk_user
POSTGRES_PASSWORD=duduk_password

# --- Django Configuration ---
SECRET_KEY=your_secret_key_here
DEBUG=True
ALLOWED_HOSTS=localhost,127.0.0.1,backend

# --- Google AI Configuration ---
GOOGLE_API_KEY=your_gemini_api_key_here

```

### 3. 실행 (Docker)

모든 서비스(DB, Backend, Frontend)를 한 번에 실행합니다.

```bash
docker-compose up --build

```

> **Note**: 초기 실행 시 PostgreSQL 데이터베이스 초기화 및 Django 마이그레이션이 자동으로 수행됩니다.

### 4. 접속 주소

* **웹 서비스**: [http://localhost:3000](https://www.google.com/search?q=http://localhost:3000)
* **API 문서**: [http://localhost:8000/api/schema](https://www.google.com/search?q=http://localhost:8000/api/schema) (Swagger/Redoc 설정 시)
* **Admin 페이지**: [http://localhost:8000/admin](https://www.google.com/search?q=http://localhost:8000/admin)

## 📂 폴더 구조 (Project Structure)

핵심 디렉토리 구조는 다음과 같습니다.

```text
duduk-project/
├── 🐳 docker-compose.yml      # 전체 서비스 오케스트레이션
├── 📂 backend/                # Django REST Framework
│   ├── 📂 apps/               # 기능별 애플리케이션
│   │   ├── 👤 users/          # 사용자, 프로필, 캐릭터
│   │   ├── 💳 transactions/   # 가계부, 예산, 지출 내역
│   │   ├── 🤖 coaching/       # AI 분석 및 피드백
│   │   └── 🏆 challenges/     # 챌린지 및 보상
│   ├── 📂 external/           # 외부 API (Gemini) 연동
│   └── 📄 requirements.txt    # Python 의존성
│
└── 📂 frontend/               # Next.js App Router
    ├── 📂 public/             # 정적 파일 (캐릭터 이미지 등)
    ├── 📂 src/
    │   ├── 📂 app/            # 페이지 라우팅 ((main), (auth))
    │   ├── 📂 components/     # 재사용 컴포넌트
    │   └── 📂 lib/            # API 클라이언트 및 유틸
    └── 📄 package.json        # Node.js 의존성

```
